Description

In this quarter, I drew a dataset of Chinese surname spelt in different Asian countries. It includes 120 common surnames of single character. Due to the pronunciation, the way they spell is not the same. So I tried to use grid to display an easy indexing view of these surnames.

Visualization library is processing.js, which is very popular among digital art communities.

Grid is a natural way to enumerate all Chinese characters at the same time. But to display the different ways of spelling, the relation between them should also be very clear and discriminative, which I do not think I have shown in an elegant way.

Result

I do not think I have conveyed the information effectively, but I want to say that it requires more knowledge to implement the transformation and multiple event handling.

There are two reasons. One is because I am a tyro in both the library and the visualization technique. The other is that I do not think processing is a good choice to display the UI-like visualization to deal with several event in the same time, because the rendering pipeline is active all the time, which may take a lot of CPU time.
